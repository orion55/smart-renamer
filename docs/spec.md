# smart-renamer Specification

## Executive Summary

TypeScript CLI утилита для автоматического переименования медиафайлов (сериалов и фильмов) с переводом названий на русский язык через YandexGPT 5.1 Pro. Улучшенная версия PowerShell скрипта `renameSerial.ps1` с расширенными возможностями нормализации и поддержкой фильмов.

## Problem Statement

**Текущая боль**: Загруженные медиафайлы имеют нечитаемые имена с техническим мусором:
- `The.Walking.Dead.S05E03.720p.BluRay.x264-DEMAND.mkv`
- `Game.of.Thrones.2011.S01E01.WEB-DL.1080p.mkv`

**Желаемый результат**: Чистые русские названия в удобном формате:
- `Ходячие мертвецы/0503.mkv`
- `Игра престолов/01.mkv`

**Почему YandexGPT вместо Translate API**:
1. Контекстный перевод — знает официальные русские названия (Кинопоиск)
2. Нормализация — удаляет rip-группы, кодеки, качество из названий
3. Гибкость — понимает нестандартные форматы именования

## Success Criteria

- [ ] Полная замена PS1-скрипта — все сценарии работают идентично
- [ ] Поддержка фильмов в дополнение к сериалам
- [ ] Использование официальных русских названий (как на Кинопоиске)
- [ ] Успешная обработка 95%+ файлов без ручного вмешательства
- [ ] Время обработки 100 файлов < 2 минут

## User Personas

**Primary**: Владелец медиатеки (1 пользователь)
- Технический уровень: средний (запуск CLI, редактирование .env)
- Цель: автоматизировать рутину переименования после скачивания
- Среда: Windows, локальная папка `f:\Сериалы`

## User Journey

1. Пользователь скачивает новые серии/фильмы в папку название, которой задаётся в файле .env в переменной IN_DIR
2. Запускает `smart-renamer`
3. CLI сканирует папку рекурсивно
4. Обнаруживает файлы, требующие переименования
5. Отправляет батчи названий в YandexGPT
6. Получает переведённые/нормализованные названия
7. Переименовывает файлы и папки
8. Выводит результат в консоль + лог-файл
9. Завершает работу с итоговой статистикой

## Functional Requirements

### Must Have (P0)

#### R1: Сканирование директории
- Рекурсивный обход всех подпапок
- Поддержка расширений: `.mkv`, `.avi`, `.mp4`, `.ts`
- Удаление лишних файлов (`.srt`, `.nfo`, `.txt`)
- **AC**: При запуске на папке с 3 уровнями вложенности находит все видеофайлы

#### R2: Определение типа контента

**Сериал** — папка или файл с маркером сезона/эпизода:
- `S01E05`, `s01e05`, `1x05` — стандартные форматы
- `S01` / `s01` отдельно (папка сезона без эпизода)
- `(Season 1)`, `(Сезон 1)`, `. Сезон 1`, `Season 2` через пробел
- `_1 sezon`, цифра в конце имени (`Ein Fall fur Zwei 1`)
- Год-диапазон `YYYY-YYYY` (`The.Booth.at.the.End.2011-2012`) — multi-season серия

**Фильм** — файл с расширением ИЛИ папка:
- Содержит год (`(2024)`, `.2024.`, `_YYYY`) без маркера сезона
- `Collection` в названии → коллекция фильмов (папка с несколькими фильмами)

**Уже обработанные** (не классифицировать, только чистить мусор или пропускать):
- Кириллическое имя — `Мёртвые лилии`, `Богомол. Сезон 1`, `Бюро легенд (Le Bureau des Légendes)`

**Без маркеров** (`dolmen`, `Schnell`, `Der Pass (HDRezka Studio)`, `JosephBalsamo.DvDrip.[rutracker.org]`) — определяется по количеству видеофайлов после удаления мусора:
- **1 видеофайл** в папке → одиночный фильм → поднять файл в IN_DIR, папку удалить
- **2+ видеофайла** в папке → серии/части → переименовать в `01`, `02`, `03`...
- Отдельный файл (не в папке) → фильм; GPT переводит название

**AC**: Корректно классифицирует реальные примеры:
- `surface.S01.rudub.web-dlrip.xvid.ac3.-hqh` → сериал
- `Diane.de.Poitiers.2022.WEBDLRip-ylnian` → фильм
- `Мёртвые лилии.2020.WEB-DL 720p.Files-x` → фильм (уже рус., нужна чистка)
- `dolmen` → сериал (папка без маркеров)

#### R3: Нормализация через YandexGPT

GPT получает «сырое» имя папки или файла и возвращает чистое русское название. Три сценария:

1. **Иностранное название** → перевести + очистить мусор
   - `surface.S01.rudub.web-dlrip.xvid.ac3.-hqh` → `Поверхность`
   - `a.knight.of.the.seven.kingdoms.s01.syncmer` → `Рыцарь семи королевств`

2. **Транслитерированное русское название** → восстановить кириллицу + очистить мусор
   - `Pochuvstvuj.moj.golos.2026.NF.WEB-DLRip.AVC` → `Почувствуй мой голос`
   - `Polozhenie.2025.WEB-DLRip.AVC` → `Положение`

3. **Уже русское, но с техническим мусором** → только очистить, не переводить
   - `Мёртвые лилии.2020.WEB-DL 720p.Files-x` → `Мёртвые лилии`
   - `Калейдоскоп (Сезон 1) HDRezka` → `Калейдоскоп`
   - `Бюро легенд (Le Bureau des Légendes)` → `Бюро легенд`
   - `Рождественская сказка.2008.BDRip 720p msltel` → `Рождественская сказка`

**AC**: `Game.of.Thrones.S01E01.720p.BluRay-LOL` → `Игра престолов`

#### R4: Переименование файлов

**Сериалы** (файлы внутри папки сериала):
- Один сезон или нет маркера сезона: `S01E05.mkv` → `05.mkv` (2 цифры)
- Несколько сезонов: `S02E05.mkv` → `0205.mkv` (4 цифры)

**Правило единообразия внутри папки**:
- Все файлы в папке должны быть в одном формате: либо все `01`/`02`/`03` (2 цифры), либо все `0101`/`0102`/`0201` (4 цифры)
- Смешение форматов недопустимо
- Формат определяется по наличию нескольких сезонов: если в папке есть файлы с разными номерами сезона (`S01...` и `S02...`) → 4 цифры; иначе → 2 цифры
- Если среди файлов уже есть `0101` (4 цифры) — привести все к 4 цифрам; если есть `01` (2 цифры) — привести все к 2 цифрам

**Фильмы — отдельные файлы в корне IN_DIR**:
- С техническим мусором: `Frenzy.1972.720p.BluRay.2xRus.Eng.HDCLUB.mkv` → `Головокружение.mkv`
- Уже русское чистое имя: `Люблю тебя до смерти.avi` → пропустить (не переименовывать)
- Транслитерация: `Pochuvstvuj.moj.golos.2026.NF.WEB-DLRip.AVC.mkv` → `Почувствуй мой голос.mkv`

**AC**: `S01E01.mkv`, `S01E02.mkv` в папке с одним сезоном → `01.mkv`, `02.mkv`; `S01E01.mkv`, `S02E01.mkv` → `0101.mkv`, `0201.mkv`; смешанный результат `01.mkv` + `0201.mkv` в одной папке недопустим

#### R5: Переименование папок

Область применения: все папки первого уровня в IN_DIR. Три сценария:

**Иностранное/транслитерированное → перевести + очистить**:
- `surface.S01.rudub.web-dlrip.xvid.ac3.-hqh` → `Поверхность`
- `The.Walking.Dead.S05.1080p.ViruseProject` → `Ходячие мертвецы`
- Маркер сезона удалять из имени папки (папка = название сериала, не сезона)

**Уже русское с мусором → только очистить**:
- `Мёртвые лилии.2020.WEB-DL 720p.Files-x` → `Мёртвые лилии`
- `Рождественская сказка.2008.BDRip 720p msltel.mkv` → `Рождественская сказка` (убрать расширение из имени папки)
- `Калейдоскоп (Сезон 1) HDRezka` → `Калейдоскоп`
- `Бюро легенд (Le Bureau des Légendes)` → `Бюро легенд`
- `Манипуляции (Сезон 1) Тайм Медиа Групп` → `Манипуляции`

**Уже чистое русское → пропустить**:
- `Амстердамская империя`, `Истоки`, `Монмартр` → без изменений

**AC**: `The.Walking.Dead.S05` → `Ходячие мертвецы`; `Мёртвые лилии.2020.WEB-DL 720p.Files-x` → `Мёртвые лилии`

#### R6: Разворачивание вложенности и определение типа по содержимому

После удаления мусорных файлов (`.srt`, `.nfo`, `.txt` и др.) считаем видеофайлы в папке:

**1 видеофайл → одиночный фильм**:
- Поднять видеофайл на уровень IN_DIR
- Переименовать файл по названию папки (через GPT если нужно), сохранить расширение
- Удалить исходную папку

**2+ видеофайла → серии/части**:
- Переименовать файлы в числовой формат: `01`, `02`, `03`... (сортировка по исходному имени)
- Папку переименовать через GPT как обычный сериал

**Вложенные папки**:
- Подъём видеофайлов из глубоких уровней в корень сериала
- Удаление пустых папок после подъёма

**AC**: Файл на 4-м уровне вложенности перемещается в корень сериала; `JosephBalsamo.DvDrip/JosephBalsamo.mkv` → `Жозеф Бальзамо.mkv` в IN_DIR

#### R7: Обработка конфликтов имён
- При существовании файла с целевым именем — удалить дубликат
- Логировать удаление
- **AC**: При конфликте остаётся существующий файл, новый удаляется

#### R8: Интеграция с YandexGPT 5.1 Pro
- Использование OpenAI SDK для Yandex Cloud SDK для Node.js
- Батчи фиксированного размера (50 файлов)
- **AC**: Успешный запрос к API возвращает переведённое название

#### R9: Обработка ошибок API
- 3 ретрая с exponential backoff (1s, 2s, 4s)
- После 3 неудач — пропуск файла, продолжение работы
- Логирование всех ошибок
- **AC**: При timeout API файл пропускается после 3 попыток

#### R10: Логирование
- Файловый лог: `logs/renamer-YYYY-MM-DD.log`
- Консольный вывод с цветами (winston + ansi-colors)
- Уровни: error, warn, info, debug
- **AC**: Лог содержит все переименования и ошибки

### Should Have (P1)

#### R11: Пропуск обработанных файлов

Объект пропускается (не отправляется в GPT), если:
- **Файл**: имя состоит только из цифр — 2-значное (`05.mkv`) или 4-значное (`0205.mkv`)
- **Папка**: имя — чистая кириллица без технических токенов и латиницы
  - Пропустить: `Амстердамская империя`, `Истоки`, `Богомол. Сезон 1`
  - Обработать: `Мёртвые лилии.2020.WEB-DL 720p.Files-x` (есть техмусор)
  - Обработать: `Калейдоскоп (Сезон 1) HDRezka` (есть группа в конце)
  - Обработать: `Бюро легенд (Le Bureau des Légendes)` (есть латиница)

Алгоритм проверки папки: удалить известные Сезон-маркеры (`Сезон N`, `Season N`) → если осталась только кириллица + пробелы → пропустить.

**AC**: `05.mkv` в папке `Игра престолов` → пропуск; `Мёртвые лилии.2020.WEB-DL 720p.Files-x` → обработать

#### R12: ASCII-баннер при старте
- cfonts баннер с названием утилиты
- Версия, дата запуска
- **AC**: При запуске отображается стилизованный баннер

#### R13: Итоговая статистика
- Количество обработанных файлов
- Количество переименований
- Количество ошибок
- Время выполнения
- **AC**: По завершении выводится сводка

### Nice to Have (P2)

#### R14: Прогресс-бар
- Визуальный индикатор прогресса обработки
- Текущий файл/папка


## Technical Architecture

### Data Model

```typescript
interface MediaFile {
  path: string;           // Полный путь к файлу
  originalName: string;   // Исходное имя файла
  extension: string;      // Расширение (.mkv, .mp4, etc.)
  type: 'series' | 'movie';
  metadata?: {
    title?: string;       // Название сериала/фильма
    season?: number;      // Номер сезона (для сериалов)
    episode?: number;     // Номер эпизода (для сериалов)
    year?: number;        // Год (для фильмов)
  };
  newName?: string;       // Новое имя после обработки
  status: 'pending' | 'processed' | 'skipped' | 'error';
}

interface GPTBatch {
  files: MediaFile[];
  prompt: string;
  response?: string;
}

interface ProcessingResult {
  total: number;
  renamed: number;
  skipped: number;
  errors: number;
  duration: number;
}
```

## Decisions (resolved)

1. **Формат промпта GPT** → **отдельные промпты на каждый сценарий**. Код заранее определяет тип через regex (перевод / транслитерация / очистка русского) и выбирает соответствующий промпт.

2. **Batch size** → **50 файлов** на запрос (быстрее и дешевле, чем 20).

3. **Rate limits** → **параллельные запросы с concurrency limit** (например, 3 одновременно) + retry с exponential backoff (R9).

4. **Детекция `s01BDRip`** → **отдать на откуп GPT**: edge-cases без разделителей не покрывать regex, пусть GPT разбирает.

5. **SDK vs HTTP** → **OpenAI SDK** с `baseURL: 'https://ai.api.cloud.yandex.net/v1'` (описано в спеке).

6. **Orphan-папка `Сезон 2`** → **поднять содержимое в родительскую папку и удалить orphan**.

7. **`Knives.Out.Collection`** → **обрабатывать как папку фильмов**: перевести название папки, сканировать внутрь и переименовывать файлы как фильмы.

8. **`JosephBalsamo.DvDrip.[rutracker.org]`** → **классификация по количеству видеофайлов после очистки** (см. R6). Год и маркеры сезона не нужны — тип определяется содержимым папки.

9. **Порядок операций** → **сначала поднять/переклассифицировать файлы (R6), затем переименовать папку (R5)**.

## Appendix: Research Findings

### YandexGPT 5.1 Pro

- **Модель**: `yandexgpt/latest` — актуальная версия 5.1 Pro
- **Контекст**: 32K токенов
- **Температура**: Рекомендуется 0.1-0.3 для детерминированных задач
- **Pricing**: ~0.0002 руб/токен (input), ~0.0006 руб/токен (output)

### Паттерны именования файлов

#### Разделители в именах

| Разделитель | Пример |
|-------------|--------|
| Точка | `The.Walking.Dead.S01E05.720p` |
| Пробел | `56 Days S01 [DUB]`, `Scarpetta S01 1080p` |
| Подчёркивание | `Crime Story_1 sezon`, `Schulmeister_1971` |
| Смешанный | `Sleeping.Dogs.2024.WEB-DLRip_от New-Team` |

#### Сериалы — маркер эпизода (файловый уровень)

- `S01E05`, `s01e05` — стандартный формат
- `1x05`, `01x05` — альтернативный формат
- `.105.`, `.0105.` — сокращённый формат

#### Сериалы — маркер сезона (папочный уровень, без эпизода)

- `S01` / `s01` — `The.Walking.Dead.S01`, `surface.S01.rudub`
- `(Season N)` — `Empathie (Season 1) WEB-DL`, `The Edge of Sleep (Season 1)`
- `Season N` через пробел — `Homicide. Life On The Street - Season 2`
- `(Сезон N)` — `Калейдоскоп (Сезон 1) HDRezka`, `Манипуляции (Сезон 1)`
- `. Сезон N` — `Богомол. Сезон 1`, `Их было десять. Сезон 1`
- `_N sezon` — `Crime Story_1 sezon`
- Цифра в конце имени — `Ein Fall fur Zwei 1`
- `(1-2.sezoni)` — диапазон сезонов: `Zhizn.(1-2.sezoni).2007-2008`

#### Фильмы

- `(2024)`, `.2024.`, `_2024_` — год в середине
- `_YYYY` — `Schulmeister_1971`
- `.YYYY` в конце — `Грозовой перевал.2026`
- `YYYY-YYYY` — год-диапазон: `The.Booth.at.the.End.2011-2012`, `Aranyelet.2015-2018`
- Отсутствие паттерна сезон/эпизод

#### Уже обработанные файлы (пропускать без GPT)

- Чисто кириллическое имя папки без технического мусора — `Амстердамская империя`, `Истоки`
- Русское название + `(Сезон N)` или `. Сезон N` — `Богомол. Сезон 1`
- Числовое имя файла (`05.mkv`, `0205.mkv`) внутри папки с русским названием

#### Транслитерированные русские названия

Требуют GPT-перевода как иностранные:
- `Pochuvstvuj.moj.golos` → Почувствуй мой голос
- `Polozhenie` → Положение
- `Zhizn` → Жизнь

#### Мусор для удаления

**Качество**: `720p`, `1080p`, `2160p`, `400p`, `4K`, `HDR`, `SDR`
