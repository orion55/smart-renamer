# smart-renamer

CLI-утилита для автоматического переименования медиафайлов (сериалов и фильмов) с переводом названий на русский язык через YandexGPT.

## Что делает

Берёт папку с загруженными файлами вида:

```
The.Walking.Dead.S05E03.720p.BluRay.x264-DEMAND.mkv
Game.of.Thrones.2011.S01E01.WEB-DL.1080p.mkv
Pochuvstvuj.moj.golos.2026.NF.WEB-DLRip.AVC/
```

И переименовывает в:

```
Ходячие мертвецы/0503.mkv
Игра престолов/01.mkv
Почувствуй мой голос.mkv
```

**Что умеет:**
- Переводит иностранные названия в официальные русские (как на Кинопоиске)
- Восстанавливает кириллицу из транслитерации (`Brigada` → `Бригада`)
- Очищает технический мусор из уже русских названий (`Слово пацана 1080p WEB-DL` → `Слово пацана`)
- Определяет тип контента: сериал / фильм / коллекция
- Нумерует эпизоды: `S01E05` → `05`, `S02E05` → `0205`
- Поднимает одиночные фильмы из папок в корень директории
- Удаляет мусорные файлы (`.srt`, `.nfo`, `.txt` и т.д.)
- Пропускает уже обработанные файлы без лишних вызовов API

## Требования

- Node.js 18+
- Аккаунт Yandex Cloud с доступом к YandexGPT API

## Установка

```bash
git clone <repo>
cd smart-renamer
npm install
cp .env.example .env
```

Заполните `.env`:

```env
YANDEX_API_KEY=y0_...   # API-ключ Yandex Cloud
IN_DIR=f:\Сериалы       # Папка с файлами для обработки
LOG_DIR=./logs          # Директория логов (опционально)
```

## Использование

```bash
# Запуск без сборки (режим разработки)
npm run dev

# Сборка и запуск бандла
npm run build
node dist/index.js
```

## Логика переименования

### Файлы сериалов

| Исходное имя | Результат | Условие |
|---|---|---|
| `S01E05.mkv` | `05.mkv` | Один сезон в папке |
| `S02E05.mkv` | `0205.mkv` | Несколько сезонов в папке |

Все файлы в папке приводятся к единому формату — смешение 2-значных и 4-значных номеров недопустимо.

### Папки без маркеров сезона/эпизода

Тип определяется по количеству видеофайлов после удаления мусора:

| Файлов | Действие |
|---|---|
| 1 | Фильм — файл поднимается в корень `IN_DIR`, папка удаляется |
| 2+ | Сериал — файлы нумеруются `01`, `02`, `03`... |

### Конфликты имён

Если файл с целевым именем уже существует — входящий дубликат удаляется, существующий файл остаётся.

## Архитектура

```
src/
├── index.ts                       # Точка входа, оркестратор
├── scanner/scanner.service.ts     # Сканирование директорий
├── classifier/classifier.service.ts # Определение типа контента
├── gpt/
│   ├── gpt.service.ts             # Клиент YandexGPT API
│   ├── prompt.builder.ts          # Формирование батч-запросов
│   └── response.parser.ts         # Разбор ответов GPT
├── renamer/
│   ├── renamer.service.ts         # Переименование файлов и папок
│   └── conflict.resolver.ts       # Разрешение конфликтов имён
├── helpers/
│   ├── patterns.ts                # Regex-паттерны
│   ├── greeting.ts                # ASCII-баннер
│   └── env.ts                     # Валидация переменных окружения
├── logger.service.ts              # Winston логгер
└── stats.ts                       # Итоговая статистика
```

**Поток данных:**
```
scan → classify → GPT translate → rename → summary
```

## Разработка

```bash
npm run check    # biome lint + format (авто-фикс)
npm run lint     # только lint
npm run format   # только format
npm run build    # сборка в dist/index.js через ncc
```

Стиль кода: одинарные кавычки, точки с запятой, отступ 2 пробела, только стрелочные функции, описательные имена переменных.

## Логи

Ротируемые логи сохраняются в `logs/renamer-YYYY-MM-DD.log`. Директория настраивается через `LOG_DIR` в `.env`.
